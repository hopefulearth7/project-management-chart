# 3.00 인원 관리 워크플로우

**워크플로우 ID**: WF-003  
**버전**: 1.0  
**작성일**: 2025-12-23  
**MLOps 적용**: 가능

---

## 개요

프로젝트에 투입되는 인원을 관리하고, 직급별 급여를 설정하며, 투입 시간을 기록하여 인건비를 자동으로 계산하는 워크플로우입니다.

---

## 워크플로우 단계

### Step 1: 직급별 급여 설정
**담당자**: 관리자  
**소요 시간**: 10분  
**입력**: 직급명, 월 급여  
**출력**: 직급별 급여 목록

**세부 사항**:
- 직급 등록/수정/삭제
- 각 직급별 기본 월 급여 입력
- 급여 적용 시작일 설정

**입력 필드**:
```
- 직급명 (필수, 예: 사원, 대리, 과장 등)
- 직급 레벨 (필수, 1~10)
- 월 급여 (필수, 0 이상)
- 적용 시작일 (필수)
- 적용 종료일 (선택, NULL이면 현재 유효)
```

**API 엔드포인트**:
```
POST /api/roles
Body: {
  name: string,
  level: number
}

POST /api/salaries
Body: {
  role_id: number,
  monthly_salary: number,
  effective_date: date,
  end_date?: date
}
```

**데이터베이스 작업**:
```sql
-- 직급 등록
INSERT INTO role (name, level)
VALUES (?, ?);

-- 급여 등록
INSERT INTO salary (role_id, monthly_salary, effective_date, end_date)
VALUES (?, ?, ?, ?);
```

**초기 직급 및 급여 예시**:
- 사원: $3,000/월
- 대리: $4,000/월
- 과장: $5,500/월
- 차장: $7,000/월
- 부장: $9,000/월
- 이사: $12,000/월

### Step 3: 투입 시간 기록
**담당자**: 사용자 / 관리자  
**소요 시간**: 매일 5분  
**입력**: 날짜, 투입 시간, 작업 내용  
**출력**: 시간 기록 목록

**세부 사항**:

**입력 필드**:

### Step 4: 인건비 자동 계산
**담당자**: 시스템 (자동)  
**소요 시간**: 즉시  
**입력**: 직급별 급여, 투입 시간  
**출력**: 인건비 총액

**세부 사항**:
- 직급별 평균 급여 적용
- 투입 시간 기반 비용 계산
- 실시간 총 인건비 표시

**계산 로직**:
```javascript
// 월 근무 시간 기준 (예: 160시간/월)
const MONTHLY_WORK_HOURS = 160;

function calculateLaborCost(member, timeEntries) {
  // 직급별 월 급여 조회
  const monthlySalary = getRoleSalary(member.role_id);
  
  // 시간당 급여 계산
  const hourlyRate = monthlySalary / MONTHLY_WORK_HOURS;
  
  // 해당 멤버의 총 투입 시간 계산
  const totalHours = timeEntries
    .filter(entry => entry.user_id === member.user_id)
    .reduce((sum, entry) => sum + entry.hours, 0);
  
  // 인건비 계산
  const laborCost = hourlyRate * totalHours;
  
  return laborCost;
}

// 프로젝트 전체 인건비 계산
function calculateProjectLaborCost(projectId) {
  const members = getProjectMembers(projectId);
  const timeEntries = getTimeEntries(projectId);
  
  let totalLaborCost = 0;
  
  members.forEach(member => {
    const memberCost = calculateLaborCost(member, timeEntries);
    totalLaborCost += memberCost;
  });
  
  return totalLaborCost;
}
```

**SQL 쿼리**:
```sql
SELECT 
  pm.project_id,
  pm.user_id,
  pm.role_id,
  s.monthly_salary,
  SUM(te.hours) as total_hours,
  -- 시간당 급여 계산 (월 160시간 기준)
  (s.monthly_salary / 160.0) as hourly_rate,
  -- 인건비 계산
  (s.monthly_salary / 160.0) * SUM(te.hours) as labor_cost
FROM project_member pm
JOIN salary s ON pm.role_id = s.role_id
LEFT JOIN time_entry te ON pm.project_id = te.project_id 
  AND pm.user_id = te.user_id
WHERE pm.project_id = ?
  AND s.end_date IS NULL
GROUP BY pm.project_id, pm.user_id, pm.role_id, s.monthly_salary;
```

**API 엔드포인트**:
```
GET /api/projects/{project_id}/labor-cost
Response: {
  total_labor_cost: number,
  member_costs: [
    {

      user_name: string,
      role_name: string,
      total_hours: number,
      hourly_rate: number,
      labor_cost: number
    },
    ...
  ]
}
```

### Step 5: 비용 업데이트 및 ROI 재계산
**담당자**: 시스템 (자동)  
**소요 시간**: 즉시  
**입력**: 계산된 인건비  
**출력**: 업데이트된 ROI

**세부 사항**:
- 계산된 인건비를 비용 테이블에 저장 또는 업데이트
- 총 비용 재계산
- ROI 자동 재계산

**인건비를 비용으로 저장**:
```sql
-- 인건비를 비용으로 저장 (또는 별도 관리)
INSERT INTO cost (
  project_id,
  cost_type,
  category,
  amount,
  description
)
SELECT 
  project_id,
  'recurring' as cost_type,
  'labor' as category,
  SUM(labor_cost) as amount,
  '인건비 (자동 계산)' as description
FROM (
  -- 위의 인건비 계산 쿼리 결과
) as labor_costs
GROUP BY project_id;
```

## 에러 처리

### 에러 시나리오

1. **직급 급여 미설정**
   - 에러: 해당 직급의 급여 정보 없음
   - 처리: 급여 설정 요청 메시지

2. **투입 시간 중복**
   - 에러: 동일 날짜에 중복 기록
   - 처리: 기존 기록 수정 또는 삭제 후 재입력

3. **인건비 계산 오류**
   - 에러: 시간 기록 데이터 누락
   - 처리: 시간 기록 요청
