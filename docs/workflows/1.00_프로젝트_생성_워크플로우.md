# 1.00 프로젝트 생성 워크플로우

**워크플로우 ID**: WF-001  
**버전**: 1.0  
**작성일**: 2025-12-23  
**MLOps 적용**: 가능

---

## 개요

프로젝트를 생성하고 기본 정보를 입력하는 워크플로우입니다. 관리자가 새로운 프로젝트를 등록하고 초기 설정을 완료합니다.

---

## 워크플로우 단계

### Step 1: 관리자 로그인
**담당자**: 관리자  
**소요 시간**: 1분  
**입력**: 이메일, 비밀번호  
**출력**: JWT 토큰

**세부 사항**:
- 관리자 계정으로 로그인
- JWT 토큰 발급 및 저장
- 세션 유지

**API 엔드포인트**:
```
POST /api/auth/login
Body: { email, password }
Response: { token, user }
```

---

### Step 2: 프로젝트 생성 화면 접근
**담당자**: 관리자  
**소요 시간**: 30초  
**입력**: 없음  
**출력**: 프로젝트 생성 폼

**세부 사항**:
- 어드민 대시보드에서 "프로젝트 생성" 버튼 클릭
- 프로젝트 생성 폼 표시

**UI 컴포넌트**:
- `ProjectCreateForm` 컴포넌트

---

### Step 3: 프로젝트 기본 정보 입력
**담당자**: 관리자  
**소요 시간**: 5분  
**입력**: 프로젝트명, 설명, 시작일, 종료일, 예상 수익  
**출력**: 프로젝트 기본 정보 객체

**세부 사항**:
- **프로젝트명** (필수): 프로젝트의 이름
- **설명** (선택): 프로젝트에 대한 상세 설명
- **시작일** (필수): 프로젝트 시작 예정일
- **종료일** (선택): 프로젝트 종료 예정일
- **예상 수익 (G)** (선택): 프로젝트에서 기대하는 재정적 이득

**유효성 검증**:
- 프로젝트명: 1~255자
- 시작일: 오늘 이후 날짜
- 종료일: 시작일 이후 날짜
- 예상 수익: 0 이상의 숫자

**API 엔드포인트**:
```
POST /api/projects
Headers: { Authorization: Bearer <token> }
Body: {
  name: string,
  description?: string,
  start_date: date,
  end_date?: date,
  expected_gain?: number
}
Response: { project_id, ... }
```

**데이터베이스 작업**:
```sql
INSERT INTO project (name, description, start_date, end_date, expected_gain, created_by, status)
VALUES (?, ?, ?, ?, ?, ?, 'planning');
```

---

### Step 4: 투입 인원 선택
**담당자**: 관리자  
**소요 시간**: 10분  
**입력**: 직급별 인원 수, 투입 기간  
**출력**: 프로젝트 멤버 목록

**세부 사항**:
- 직급별로 투입할 인원 수 선택
- 각 인원의 투입 기간 설정 (시작일, 종료일)
- 직급별 평균 급여 자동 적용

**입력 필드**:
- 직급 선택 (드롭다운)
- 인원 수 (숫자 입력)
- 투입 시작일 (날짜 선택)
- 투입 종료일 (날짜 선택)

**계산 로직**:
```
인건비 = (직급별 월 급여 / 월 근무 시간) × 투입 시간 × 인원 수
```

**API 엔드포인트**:
```
POST /api/projects/{project_id}/members
Body: {
  role_id: number,
  user_ids: number[],
  start_date: date,
  end_date: date
}
```

**데이터베이스 작업**:
```sql
INSERT INTO project_member (project_id, user_id, role_id, start_date, end_date)
VALUES (?, ?, ?, ?, ?);
```

---

### Step 5: 비용 정보 입력
**담당자**: 관리자  
**소요 시간**: 15분  
**입력**: 일회성 비용, 지속적 비용  
**출력**: 비용 목록

**세부 사항**:
- **일회성 비용**: 초기 개발 비용, 인프라 구축 비용 등
- **지속적 비용**: 월간 서버 운영 비용, 라이선스 비용 등
- 각 비용 항목별 카테고리 선택

**입력 필드**:
- 비용 유형 (일회성/지속적/장기)
- 카테고리 (인프라/라이선스/인건비/기타)
- 금액
- 통화 (기본: USD)
- 적용 기간 (지속적 비용인 경우)

**API 엔드포인트**:
```
POST /api/projects/{project_id}/costs
Body: {
  cost_type: 'one-time' | 'recurring' | 'long-term',
  category: string,
  amount: number,
  currency: string,
  start_date?: date,
  end_date?: date,
  recurrence?: 'monthly' | 'yearly'
}
```

**데이터베이스 작업**:
```sql
INSERT INTO cost (project_id, cost_type, category, amount, currency, start_date, end_date, recurrence, created_by)
VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?);
```

### Step 6: ROI 계산 및 확인
**담당자**: 시스템 (자동)  
**소요 시간**: 즉시  
**입력**: 총 비용 (C), 예상 수익 (G)  
**출력**: ROI 퍼센트

**세부 사항**:
- 입력된 모든 비용을 합산하여 총 비용 (C) 계산
- 프로젝트의 예상 수익 (G) 사용
- ROI 공식 적용: `(G - C) / C × 100`

**계산 로직**:
```javascript
// 총 비용 계산
const totalCost = 
  oneTimeCosts.reduce((sum, cost) => sum + cost.amount, 0) +
  recurringCosts.reduce((sum, cost) => {
    const months = calculateMonths(cost.start_date, cost.end_date);
    return sum + (cost.amount * months);
  }, 0) +
  laborCosts.reduce((sum, member) => {
    const months = calculateMonths(member.start_date, member.end_date);
    const salary = getRoleSalary(member.role_id);
    return sum + (salary * months * member.count);
  }, 0);

// ROI 계산
const expectedGain = project.expected_gain || 0;
const roi = ((expectedGain - totalCost) / totalCost) * 100;

// ROI 이력 저장
await saveROICalculation(project_id, totalCost, expectedGain, roi);
```

**API 엔드포인트**:
```
GET /api/projects/{project_id}/roi
Response: {
  total_cost: number,
  expected_gain: number,
  roi_percentage: number,
  calculation_date: timestamp
}
```

**데이터베이스 작업**:
```sql
-- 총 비용 계산 (뷰 또는 함수 사용)
SELECT 
  project_id,
  SUM(CASE WHEN cost_type = 'one-time' THEN amount ELSE 0 END) as one_time_cost,
  SUM(CASE WHEN cost_type = 'recurring' THEN amount * recurrence_months ELSE 0 END) as recurring_cost,
  -- 인건비 계산 생략 (별도 쿼리 필요)
FROM cost

GROUP BY project_id;
```

## 에러 처리

### 에러 시나리오

1. **로그인 실패**
   - 에러: 잘못된 이메일/비밀번호
   - 처리: 에러 메시지 표시, 재시도 가능

2. **프로젝트명 중복**
   - 에러: 동일한 프로젝트명 존재
   - 처리: 중복 확인 메시지, 프로젝트명 수정 요청

3. **날짜 유효성 오류**
   - 에러: 종료일이 시작일보다 이전
   - 처리: 날짜 재선택 요청

4. **ROI 계산 오류**
   - 에러: 비용 데이터 누락
   - 처리: 필수 비용 입력 요청
