# 2.00 비용 관리 워크플로우

**워크플로우 ID**: WF-002  
**버전**: 1.0  
**작성일**: 2025-12-23  
**MLOps 적용**: 가능

---

## 개요

프로젝트의 비용을 관리하고 ROI를 자동으로 재계산하는 워크플로우입니다. 일회성 비용, 지속적 비용, 장기적 비용을 입력하고 관리합니다.

---

## 워크플로우 단계

### Step 1: 비용 관리 화면 접근
**담당자**: 관리자  
**소요 시간**: 30초  
**입력**: 프로젝트 선택  
**출력**: 비용 관리 대시보드

**세부 사항**:
- 어드민 대시보드에서 프로젝트 선택
- "비용 관리" 메뉴 클릭
- 현재 비용 현황 표시

**UI 컴포넌트**:
- `CostManagementDashboard` 컴포넌트

### Step 2: 일회성 비용 입력
**담당자**: 관리자  
**소요 시간**: 10분  
**입력**: 비용 항목, 금액, 카테고리  
**출력**: 일회성 비용 목록

**세부 사항**:
- **비용 항목명**: 예) "초기 개발 비용", "서버 구축 비용"
- **금액**: 숫자 입력
- **카테고리**: 인프라/라이선스/인건비/기타
- **설명**: 비용에 대한 상세 설명 (선택)

**입력 필드**:
```
- 비용 항목명 (필수)
- 금액 (필수, 0 이상)
- 통화 (기본: USD)
- 카테고리 (필수)
- 설명 (선택)
- 발생일 (기본: 오늘)
```

**API 엔드포인트**:
```
POST /api/projects/{project_id}/costs
Body: {
  cost_type: 'one-time',
  category: string,
  amount: number,
  currency: string,
  description?: string,
  occurred_date: date
}
```

**데이터베이스 작업**:
```sql

  project_id,

  category,

  currency,

VALUES (?, 'one-time', ?, ?, ?, ?, ?, ?);
```

**일회성 비용 예시**:
- 초기 개발 비용: $6,000
- 서버 구축 비용: $2,000
- 라이선스 구매: $1,500

### Step 3: 지속적 비용 입력
**담당자**: 관리자  
**소요 시간**: 15분  
**입력**: 비용 항목, 금액, 반복 주기  
**출력**: 지속적 비용 목록

**세부 사항**:
- **비용 항목명**: 예) "월간 서버 운영비", "연간 라이선스 비용"
- **금액**: 월간 또는 연간 금액
- **반복 주기**: 월간(monthly) / 연간(yearly)
- **적용 기간**: 시작일 ~ 종료일

**입력 필드**:
```
- 비용 항목명 (필수)
- 금액 (필수, 0 이상)
- 반복 주기 (필수: monthly/yearly)
- 적용 시작일 (필수)
- 적용 종료일 (선택, NULL이면 무기한)
- 통화 (기본: USD)
- 카테고리 (필수)
- 설명 (선택)
```

**계산 로직**:
```javascript
// 월간 비용인 경우
if (recurrence === 'monthly') {
  const months = calculateMonths(startDate, endDate);
  totalRecurringCost = amount * months;
}

// 연간 비용인 경우
if (recurrence === 'yearly') {
  const years = calculateYears(startDate, endDate);
  totalRecurringCost = amount * years;
}
```

  description?: string

**데이터베이스 작업**:

**지속적 비용 예시**:

### Step 4: 장기적 비용 계산
**담당자**: 관리자  
**소요 시간**: 5분  
**입력**: 예측 기간, 비용 증가율  
**출력**: 연도별 예상 비용 그래프

**세부 사항**:
- **예측 기간**: 3년, 5년 등
- **비용 증가율**: 연간 비용 증가율 (%)
- 현재 비용을 기반으로 장기 예측

**계산 로직**:
```javascript
function calculateLongTermCost(baseCost, years, growthRate) {
  const costs = [];
  let currentCost = baseCost;
  
  for (let year = 1; year <= years; year++) {
    currentCost = currentCost * (1 + growthRate / 100);
    costs.push({
      year: year,
      cost: currentCost,
      cumulative: costs.reduce((sum, c) => sum + c.cost, 0) + currentCost
    });
  }
  
  return costs;
}
```

**API 엔드포인트**:
```
POST /api/projects/{project_id}/costs/long-term
Body: {
  prediction_years: number,
  growth_rate: number
}
Response: {
  yearly_costs: [
    { year: 1, cost: number, cumulative: number },
    ...
  ]
}
```

**데이터베이스 작업**:
```sql

```

### Step 5: ROI 자동 계산
**담당자**: 시스템 (자동)  
**소요 시간**: 즉시  
**입력**: 총 비용 (C), 예상 수익 (G)  
**출력**: ROI 퍼센트, 계산 이력

**세부 사항**:
- 모든 비용을 집계하여 총 비용 (C) 계산
- 프로젝트의 예상 수익 (G) 사용
- ROI 공식 적용: `(G - C) / C × 100`
- 계산 결과 저장 및 이력 관리

**집계 쿼리**:
```sql
WITH cost_summary AS (
  SELECT 
    project_id,
    -- 일회성 비용 합계
    SUM(CASE WHEN cost_type = 'one-time' THEN amount ELSE 0 END) as one_time_total,
    -- 지속적 비용 합계 (월간)
    SUM(CASE 
      WHEN cost_type = 'recurring' AND recurrence = 'monthly' 
      THEN amount * EXTRACT(MONTH FROM AGE(COALESCE(end_date, CURRENT_DATE), start_date))
      ELSE 0 
    END) as recurring_monthly_total,
    -- 지속적 비용 합계 (연간)
    SUM(CASE 
      WHEN cost_type = 'recurring' AND recurrence = 'yearly' 
      THEN amount * EXTRACT(YEAR FROM AGE(COALESCE(end_date, CURRENT_DATE), start_date))
      ELSE 0 
    END) as recurring_yearly_total
  FROM cost
  WHERE project_id = ?
  GROUP BY project_id
),
labor_cost AS (
  -- 인건비 계산 (별도 쿼리)
  SELECT 
    pm.project_id,
    SUM(s.monthly_salary * EXTRACT(MONTH FROM AGE(pm.end_date, pm.start_date))) as total_labor_cost
  FROM project_member pm
  JOIN salary s ON pm.role_id = s.role_id
  WHERE pm.project_id = ?
    AND s.end_date IS NULL
  GROUP BY pm.project_id
)
SELECT 
  cs.project_id,
  cs.one_time_total + cs.recurring_monthly_total + cs.recurring_yearly_total + COALESCE(lc.total_labor_cost, 0) as total_cost,
  p.expected_gain,
  CASE 
    WHEN (cs.one_time_total + cs.recurring_monthly_total + cs.recurring_yearly_total + COALESCE(lc.total_labor_cost, 0)) > 0
    THEN ((p.expected_gain - (cs.one_time_total + cs.recurring_monthly_total + cs.recurring_yearly_total + COALESCE(lc.total_labor_cost, 0))) 
          / (cs.one_time_total + cs.recurring_monthly_total + cs.recurring_yearly_total + COALESCE(lc.total_labor_cost, 0))) * 100
    ELSE 0
  END as roi_percentage
FROM cost_summary cs
JOIN project p ON cs.project_id = p.id
LEFT JOIN labor_cost lc ON cs.project_id = lc.project_id;
```

**API 엔드포인트**:
```
POST /api/projects/{project_id}/roi/calculate
Response: {
  total_cost: number,
  expected_gain: number,
  roi_percentage: number,
  calculation_date: timestamp,
  cost_breakdown: {
    one_time: number,
    recurring: number,
    labor: number,
    long_term: number
  }
}
```

**ROI 계산 결과 저장**:
```sql

)

```

- **규칙**:

  - 날짜 유효성 검증
  - 중복 비용 방지

## 에러 처리

### 에러 시나리오

1. **비용 입력 오류**
   - 에러: 금액이 음수
   - 처리: 유효성 검증 메시지 표시

2. **날짜 범위 오류**
   - 에러: 종료일이 시작일보다 이전
   - 처리: 날짜 재선택 요청

3. **ROI 계산 실패**
   - 에러: 비용 데이터 누락
   - 처리: 필수 비용 입력 요청
