# 간트 차트 사용 가이드

## 개요

프로젝트 관리 시스템의 간트 차트는 작업 일정을 시각적으로 표시하고 관리할 수 있는 기능입니다.

## 주요 기능

### 1. 좌측 작업 정보 고정

- **코드**: 작업 코드 (예: 1.0, 1.1)
- **작업명**: 작업 이름 (클릭 가능)
- **작업자**: 할당된 담당자
- **시작일**: 작업 시작 날짜
- **종료일**: 작업 종료 날짜

좌측 열은 가로 스크롤 시에도 **고정(Sticky)**되어 항상 보입니다.

### 2. 상태별 색상 구분

간트 차트 바의 색상으로 작업 상태를 한눈에 파악할 수 있습니다:

| 색상 | 의미 | 설명 |
|------|------|------|
| 🔴 **빨간색** | 진행 중 | 시작일과 종료일이 설정되었으나 아직 완료되지 않은 작업 |
| 🔵 **파란색** | 완료 | 작업이 완료됨 |
| 🟡 **노란색** | 기간 연장 | 원래 계획된 종료일보다 기간이 연장된 작업 |
| ⚫ **회색** | 대기 | 아직 시작하지 않은 작업 |

### 3. 타임라인 표시

- **Phase 단위**: 주차별로 구분
- **Week 단위**: 주차 번호 표시
- **날짜 단위**: 요일과 날짜(월/일) 표시
- **주말 제외**: 평일만 표시 (월~금)

### 4. 인터랙티브 기능

#### 작업명 클릭
- 작업명을 클릭하면 작업 상세 정보를 볼 수 있습니다
- 작업 수정 및 상태 변경이 가능합니다

#### 간트 차트 바 클릭
- 타임라인의 색상 바를 클릭하면 작업 정보를 확인할 수 있습니다
- 마우스를 올리면 툴팁으로 상세 정보가 표시됩니다:
  - 작업명
  - 상태
  - 기간 (시작일 ~ 종료일)
  - 연장 여부 (⚠️ 표시)

## 설치 및 설정

### 1. 데이터베이스 마이그레이션

기간 연장 추적 기능을 사용하려면 데이터베이스에 `original_end_date` 필드를 추가해야 합니다:

```powershell
# PowerShell에서 실행
.\update-wbs-schema.ps1
```

또는 직접 SQL을 실행:

```powershell
psql $env:DATABASE_URL -f database/add-wbs-original-date.sql
```

### 2. 백엔드 재시작

마이그레이션 후 백엔드 서버를 재시작합니다:

```powershell
.\restart-backend.ps1
```

### 3. 프론트엔드 확인

브라우저에서 프로젝트 상세 페이지로 이동하여 간트 차트를 확인합니다.

## 사용 예시

### 작업 일정 변경 시

1. **첫 작업 생성 시**
   - `start_date`: 2024-01-01
   - `end_date`: 2024-01-10
   - `original_end_date`: (자동으로 NULL)
   - → 빨간색 바로 표시 (진행 중)

2. **작업 완료 시**
   - `status`: 'in-progress' → 'completed'
   - → 파란색 바로 변경 (완료)

3. **기간 연장 시**
   - 첫 `end_date` 변경 시, `original_end_date`에 기존 날짜 자동 저장
   - `end_date`: 2024-01-10 → 2024-01-15 변경
   - `original_end_date`: 2024-01-10 (자동 저장)
   - → 노란색 바로 변경 (연장)

## 기술 상세

### 프론트엔드 (GanttChart.jsx)

```javascript
// 색상 구분 로직
const isExtended = item.original_end_date && 
                   item.end_date && 
                   new Date(item.original_end_date) < new Date(item.end_date)

const statusColor = 
  item.status === 'completed' ? 'bg-blue-500' :  // 완료: 파란색
  isExtended ? 'bg-yellow-500' :                  // 연장: 노란색
  item.status === 'in-progress' ? 'bg-red-500' :  // 진행중: 빨간색
  'bg-gray-400'                                   // 기타: 회색
```

### 백엔드 (server.js)

```javascript
// WBS 업데이트 시 original_end_date 자동 처리
if (end_date && currentData.end_date && !original_end_date) {
  original_end_date = currentData.end_date;
}
```

### 데이터베이스 (wbs 테이블)

```sql
-- original_end_date 컬럼 추가
ALTER TABLE wbs 
ADD COLUMN IF NOT EXISTS original_end_date DATE;

-- 인덱스 추가 (성능 최적화)
CREATE INDEX IF NOT EXISTS idx_wbs_original_end_date ON wbs(original_end_date);
```

## 주의사항

1. **주말 제외**: 간트 차트는 기본적으로 주말(토, 일)을 제외하고 표시합니다.

2. **기간 연장 추적**: `original_end_date`는 첫 번째 `end_date` 변경 시에만 자동 저장됩니다. 
   이후 변경에서는 `original_end_date`가 유지됩니다.

3. **타임라인 범위**: 프로젝트의 `start_date`와 `end_date` 범위 내에서만 타임라인이 표시됩니다.

4. **브라우저 호환성**: 
   - Sticky positioning: Chrome 56+, Firefox 59+, Safari 13+
   - 구형 브라우저에서는 좌측 열이 고정되지 않을 수 있습니다.

## 문제 해결

### 간트 차트가 표시되지 않는 경우

1. 프로젝트에 `start_date`와 `end_date`가 설정되어 있는지 확인
2. WBS 작업에 `start_date`와 `end_date`가 설정되어 있는지 확인
3. 브라우저 콘솔에서 에러 메시지 확인

### 색상이 제대로 표시되지 않는 경우

1. 데이터베이스 마이그레이션이 완료되었는지 확인:
   ```sql
   SELECT column_name FROM information_schema.columns 
   WHERE table_name = 'wbs' AND column_name = 'original_end_date';
   ```

2. 백엔드가 최신 코드로 재시작되었는지 확인

3. 브라우저 캐시 삭제 후 새로고침 (Ctrl + Shift + R)

### 좌측 열이 고정되지 않는 경우

- 브라우저 버전을 최신으로 업데이트
- 다른 브라우저에서 테스트 (Chrome, Firefox, Edge 등)
