# PRD (Product Requirements Document)
## 프로젝트 관리 웹사이트

**버전**: 1.0  
**작성일**: 2025-12-23  
**작성자**: 개발팀  
**프로젝트 단계**: PoC (Proof of Concept)

---

## 1. 프로젝트 개요

### 1.1 목적
프로젝트의 ROI(Return on Investment)를 계산하고 관리하기 위한 웹 기반 시스템을 구축합니다. 프로젝트 비용, 투입 인원, 시간 등을 관리하여 투자 대비 수익률을 실시간으로 모니터링할 수 있습니다.

### 1.2 범위
- 로컬 웹 환경에서 동작하는 Front 및 Admin 페이지
- 로컬 데이터베이스 연동
- 회원가입 및 로그인 기능
- 프로젝트 비용 관리 및 ROI 계산
- 투입 인원 및 시간 관리

### 1.3 기술 스택 (PoC)
- **Frontend**: React / Vue.js / Next.js (로컬 웹)
- **Backend**: Node.js (Express) / Python (FastAPI) (로컬 서버)
- **Database**: PostgreSQL / MySQL (로컬 설치)
- **인증**: JWT 기반 인증

## 2. 기능 요구사항

### 2.1 인증 기능 (Authentication)

#### 2.1.1 회원가입
- 이메일, 비밀번호, 이름 입력
- 비밀번호 유효성 검증 (최소 8자, 영문+숫자+특수문자)
- 이메일 중복 확인
- 역할 선택 (일반 사용자 / 관리자)

#### 2.1.2 로그인
- 이메일/비밀번호 기반 로그인
- JWT 토큰 발급
- 세션 관리
- 자동 로그인 기능 (선택사항)

### 2.2 어드민 페이지 (Admin)

#### 2.2.1 직급별 월 급여 관리
- 직급 등록/수정/삭제
- 직급별 기본 월 급여 입력
- 직급 목록: 사원, 대리, 과장, 차장, 부장, 이사 등
- 급여 이력 관리

#### 2.2.2 프로젝트 투입 시간 관리
- 프로젝트별 투입 시간 입력
- 인원별 투입 시간 기록
- 시간 단위: 시간(Hour) 또는 일(Day)
- 프로젝트 기간 설정

#### 2.2.3 비용 관리
**일회성 비용 (One-time Cost)**
- 초기 개발 비용
- 인프라 구축 비용
- 라이선스 구매 비용
- 기타 일회성 지출

**지속적 비용 (Recurring Cost)**
- 월간 서버 운영 비용
- 월간 라이선스 비용
- 월간 인프라 비용
- 기타 정기적 지출

**장기적 비용 (Long-term Cost)**
- 연간 총 비용 계산
- 프로젝트 기간별 누적 비용
- 예상 총 비용 산출

#### 2.2.4 ROI 계산
**계산 공식**: `(G - C) / C = ROI`

- **G (Gain)**: 프로젝트에서 기대하는 재정적 이득
- **C (Cost)**: 프로젝트에 투자하는 초기 비용 + 지속적인 비용

**계산 예시**:
```
초기 비용: $6,000
월 지속 비용: $25 × 12개월 = $300
총 비용 (C): $6,300
예상 수익 (G): $10,000

ROI 계산:
($10,000 - $6,300) ÷ $6,300 = 0.5873
ROI = 0.5873 × 100 = 58.7%
```

**기능**:
- 실시간 ROI 계산
- ROI 그래프 시각화
- ROI 이력 추적
- 목표 ROI 설정 및 알림

### 2.3 프론트 페이지 (Front)

#### 2.3.1 투입 인원 선택
- 프로젝트별 투입 인원 선택
- 인원별 직급 선택
- 투입 기간 설정
- 인원 추가/제거 기능

#### 2.3.2 직급별 투입 금액 계산
- 선택한 인원의 직급별 평균 급여 적용
- 투입 시간 기반 비용 계산
- 실시간 총 비용 표시
- 비용 상세 내역 보기

**계산 로직**:
```
인원 비용 = (직급별 월 급여 / 월 근무 시간) × 투입 시간 × 인원 수
총 인건비 = Σ(각 인원별 비용)
```

#### 2.3.3 만족도 표시
- 프로젝트별 만족도 평가 (1~5점)
- 만족도 그래프 시각화
- 만족도 이력 추적
- 만족도 기반 프로젝트 필터링

## 3. 비기능 요구사항

### 3.1 성능
- 페이지 로딩 시간: 2초 이내
- API 응답 시간: 500ms 이내
- 동시 사용자: 50명 이상 지원

### 3.2 보안
- 비밀번호 암호화 (bcrypt)
- JWT 토큰 기반 인증
- SQL Injection 방지
- XSS 공격 방지
- CSRF 토큰 적용

### 3.3 사용성
- 반응형 디자인 (모바일, 태블릿, 데스크톱)
- 직관적인 UI/UX
- 접근성 고려 (WCAG 2.1 Level AA)

### 3.4 호환성
- Chrome, Firefox, Safari, Edge 최신 버전 지원
- 로컬 환경에서 동작
