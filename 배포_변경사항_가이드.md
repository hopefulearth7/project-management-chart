## 🚀 배포 방법

### 방법 1: 자동 배포 스크립트 사용 (권장)

```powershell
# 배포 스크립트 실행
.\deploy-changes.ps1
```

스크립트가 자동으로:
1. 변경사항 확인
2. Git에 추가 및 커밋
3. 원격 저장소로 푸시
4. Render 자동 배포 트리거

### 방법 2: 수동 배포

```bash
# 1. 변경사항 확인
git status

# 2. 변경사항 추가
git add .

# 3. 커밋
git commit -m "feat: 직급 생성 기능 추가 및 UI 개선"

# 4. 원격 저장소로 푸시 (Render가 자동으로 배포)
git push origin main
```

## ⚙️ Render 자동 배포 설정

Render는 기본적으로 GitHub 저장소에 푸시할 때 자동으로 배포됩니다.

### 자동 배포가 작동하지 않는 경우

1. **Render 대시보드** → 서비스 선택
2. **Settings** 탭
3. **Auto-Deploy** 섹션 확인:
   - ✅ "Yes"로 설정되어 있어야 함
   - Branch: `main` (또는 사용 중인 브랜치)

### 수동 재배포

1. **Render 대시보드** → 서비스 선택
2. **Manual Deploy** 버튼 클릭
3. 브랜치 선택 후 "Deploy latest commit" 클릭

## 🔧 문제 해결

### 배포가 실패하는 경우

1. **로그 확인**
   - Render 대시보드 → 서비스 → **Logs** 탭
   - 에러 메시지 확인

2. **일반적인 문제**
   - 빌드 오류: `package.json` 확인
   - 환경 변수 누락: **Environment** 탭 확인
   - 데이터베이스 연결 오류: `DATABASE_URL` 확인

### 변경사항이 반영되지 않는 경우

1. **캐시 문제**
   ```bash
   # 프론트엔드 빌드 캐시 클리어
   cd frontend
   rm -rf dist node_modules/.vite
   npm run build
   ```

2. **브라우저 캐시 클리어**
   - Ctrl + Shift + R (강력 새로고침)
   - 또는 브라우저 개발자 도구 → Network → "Disable cache" 체크
