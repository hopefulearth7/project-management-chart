## 🚀 시작하기

### 시스템 요구사항

- **Node.js**: 18.x 이상
- **PostgreSQL**: 15.x 이상
- **RAM**: 최소 4GB
- **OS**: Windows 10/11, macOS, Linux

### 5분 빠른 시작

```powershell
# 1. 환경 설정
.\create-env.ps1

# 2. 완전 설치 (데이터베이스 + 의존성)
.\setup-complete.ps1

# 3. 개발 서버 실행
.\start-dev.ps1
```

브라우저에서 http://localhost:5173 접속 → 완료! 🎉

## 📦 설치 가이드

### 단계 1: 저장소 클론

```bash
git clone <repository-url>
cd project_manage
```

### 단계 2: PostgreSQL 설정

#### 방법 A: 자동 설정 (권장)

```powershell
.\setup-postgres-user.ps1
```

#### 방법 B: 수동 설정

```sql
-- PostgreSQL에 접속
psql -U postgres

-- 데이터베이스 생성
CREATE DATABASE project_management;
CREATE USER pm_user WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE project_management TO pm_user;
```

### 단계 3: 환경 변수 설정

```powershell
.\create-env.ps1
```

`.env` 파일이 생성되며, 다음 내용이 포함됩니다:

```env
PORT=3000
DATABASE_URL=postgresql://pm_user:password@localhost:5432/project_management
JWT_SECRET=your_jwt_secret
CORS_ORIGIN=http://localhost:5173
NODE_ENV=development
```

### 단계 4: 데이터베이스 초기화

```powershell
# 스키마 생성 + 초기 데이터
.\setup-complete.ps1
```

### 단계 5: 서버 실행

```powershell
# 백엔드 + 프론트엔드 동시 실행
.\start-dev.ps1
```

## 📊 간트 차트 사용법

### 기본 사용

1. **프로젝트 상세 페이지** 이동
2. **WBS** 탭 선택
3. **작업 추가** 버튼 클릭
4. 작업 정보 입력:
   - 작업명
   - 시작일 / 종료일
   - 담당자
   - 상태 (not-started, in-progress, completed)

### 색상 구분

| 색상 | 의미 | 설명 |
|------|------|------|
| 🔴 **빨간색** | 진행 중 | 시작일~종료일이 설정되었으나 미완료 |
| 🔵 **파란색** | 완료 | 작업이 완료됨 |
| 🟡 **노란색** | 기간 연장 | 원래 계획보다 기간이 늘어남 |
| ⚫ **회색** | 대기 | 아직 시작하지 않음 |

### 좌측 열 고정

간트 차트를 좌우로 스크롤해도 **코드, 작업명, 작업자, 시작일, 종료일** 열은 항상 고정되어 보입니다.

### 인터랙티브 기능

- **작업명 클릭**: 작업 상세 정보 확인
- **간트 바 클릭**: 일정 수정
- **마우스 오버**: 툴팁으로 상세 정보 표시

자세한 내용은 [`docs/간트차트_사용가이드.md`](docs/간트차트_사용가이드.md) 참조

## 🔧 문제 해결

### 백엔드가 시작되지 않는 경우

```powershell
# 1단계: 데이터베이스 연결 확인
.\check-db.ps1

# 2단계: 포트 충돌 해결
.\kill-backend.ps1

# 3단계: 백엔드 재시작
.\restart-backend.ps1
```

### 데이터베이스 연결 오류

```powershell
# .env 파일 확인
cat .env

# DATABASE_URL 형식 확인
# postgresql://사용자:비밀번호@호스트:포트/데이터베이스명

# PostgreSQL 서비스 확인
Get-Service -Name postgresql*
```

### CORS 오류

`.env` 파일에서 `CORS_ORIGIN` 확인:

```env
CORS_ORIGIN=http://localhost:5173
```

### 간트 차트가 표시되지 않는 경우

```powershell
# WBS 스키마 업데이트
.\update-wbs-schema.ps1

# 백엔드 재시작
.\restart-backend.ps1

# 브라우저 캐시 삭제 (Ctrl + Shift + R)
```

### API가 응답하지 않는 경우

```powershell
# 백엔드 로그 확인
# 백엔드 터미널에서 에러 메시지 확인

# API 테스트
.\test-all-apis.ps1
```
