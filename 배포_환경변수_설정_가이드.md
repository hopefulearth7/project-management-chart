# 🌐 배포 환경 변수 설정 가이드

## ✅ 자동 설정 완료

코드가 자동으로 배포 환경을 감지하여 배포된 백엔드 URL을 사용하도록 설정되었습니다.

- **프론트엔드**: https://project-management-frontend-ogk2.onrender.com
- **백엔드**: https://project-management-backend-vn80.onrender.com

## 📋 Render 환경 변수 설정 (선택사항)

환경 변수를 명시적으로 설정하려면:

### 프론트엔드 환경 변수 (Render)

Render 대시보드 > 프론트엔드 서비스 > Environment

```
VITE_API_URL=https://project-management-backend-vn80.onrender.com
```

### 백엔드 환경 변수 (Render)

Render 대시보드 > 백엔드 서비스 > Environment

```
PORT=3000
DATABASE_URL=<데이터베이스 연결 문자열>
CORS_ORIGIN=https://project-management-frontend-ogk2.onrender.com,http://localhost:5173
NODE_ENV=production
JWT_SECRET=<랜덤 시크릿 키>
```

## 🔄 변경 사항

### 1. 프론트엔드 (`frontend/src/utils/api.js`)

- ✅ 배포 환경 자동 감지
- ✅ 배포된 백엔드 URL 자동 사용
- ✅ localhost 환경에서는 Vite proxy 사용 (기존 동작 유지)

### 2. 백엔드 (`backend/server.js`)

- ✅ CORS 설정 개선: 여러 origin 허용
- ✅ 배포된 프론트엔드 URL 자동 허용
- ✅ 개발 환경 localhost 허용

## 🧪 테스트

### 배포 사이트에서 테스트

1. https://project-management-frontend-ogk2.onrender.com 접속
2. 브라우저 개발자 도구(F12) > Console 확인
3. `🌐 배포 환경 감지: 배포된 백엔드 API 사용` 메시지 확인
4. 모든 기능이 정상 작동하는지 확인

### 로컬 개발 환경

로컬에서는 기존과 동일하게 동작:
- `npm run dev` 실행
- `http://localhost:5173` 접속
- Vite proxy를 통해 `http://localhost:3000`으로 요청

## 📝 참고

- 환경 변수를 설정하지 않아도 자동으로 배포된 백엔드를 사용합니다.
- 명시적으로 설정하려면 위의 환경 변수를 Render 대시보드에 추가하세요.
- 로컬 개발 환경에는 영향을 주지 않습니다.
