# 🧪 배포된 사이트 테스트 가이드

## ✅ 자동 테스트 스크립트

```powershell
.\test-deployed-site.ps1
```

이 스크립트는 다음을 테스트합니다:
1. 백엔드 Health Check API
2. 프론트엔드 서버 상태
3. 관리자 설정 API
4. 관리자 코드 검증 API

## 🌐 배포된 사이트 URL

- **프론트엔드**: https://project-management-frontend-ogk2.onrender.com
- **백엔드 API**: https://project-management-backend-vn80.onrender.com

## 📋 수동 테스트 방법

### 1. 프론트엔드 접속 테스트

브라우저에서 접속:
```
https://project-management-frontend-ogk2.onrender.com
```

**확인 사항:**
- ✅ 페이지가 정상적으로 로드되는지
- ✅ 브라우저 콘솔에서 `🌐 배포 환경 감지: 배포된 백엔드 API 사용` 메시지 확인
- ✅ 네트워크 탭에서 API 요청이 `https://project-management-backend-vn80.onrender.com/api/...`로 가는지 확인

### 2. 백엔드 API 테스트

브라우저 또는 curl로 테스트:

```bash
# Health Check
curl https://project-management-backend-vn80.onrender.com/api/health

# 관리자 설정 조회
curl https://project-management-backend-vn80.onrender.com/api/admin/settings

# 관리자 코드 검증
curl -X POST https://project-management-backend-vn80.onrender.com/api/admin/verify-code \
  -H "Content-Type: application/json" \
  -d '{"code":"test","company_name":"test"}'
```

### 3. 기능 테스트

1. **프로젝트 목록 확인**
   - https://project-management-frontend-ogk2.onrender.com/projects 접속
   - 프로젝트 목록이 정상적으로 표시되는지 확인

2. **프로젝트 상세 페이지**
   - https://project-management-frontend-ogk2.onrender.com/projects/30 접속
   - 프로젝트 정보가 정상적으로 표시되는지 확인

3. **관리자 페이지**
   - 관리자 메뉴 클릭
   - 관리자 코드와 회사명 입력
   - 관리자 페이지가 정상적으로 표시되는지 확인

## ⚠️ 주의사항

### Render 무료 플랜 제한

- **15분 비활성 시 자동 정지**
- 첫 접속 시 **최대 50초** 소요 (Cold Start)
- 테스트 전에 서버를 깨워야 할 수 있음

### 서버 깨우기

```bash
curl https://project-management-backend-vn80.onrender.com/api/health
```

또는 브라우저에서 백엔드 URL에 접속하여 서버를 깨움

## 🔍 문제 해결

### API 요청이 localhost로 가는 경우

1. 브라우저 캐시 삭제 (Ctrl + Shift + Delete)
2. 하드 새로고침 (Ctrl + F5)
3. 브라우저 콘솔 확인
4. 네트워크 탭에서 실제 요청 URL 확인

### CORS 오류가 발생하는 경우

1. 백엔드 CORS 설정 확인
2. Render 환경 변수에서 `CORS_ORIGIN` 확인
3. 프론트엔드 URL이 CORS 허용 목록에 있는지 확인
